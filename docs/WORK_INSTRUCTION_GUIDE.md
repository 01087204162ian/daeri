# 작업 지시 시작 가이드

> **목적**: 작업을 시작할 때 효율적이고 명확하게 지시하는 방법 가이드  
> **대상**: 사용자가 AI Assistant에게 작업을 지시할 때

---

## 🎯 작업 지시 시작 방법 (권장)

### 방법 1: 단계별 지시 (권장) ⭐

**새로운 작업이나 복잡한 작업일 때**

```
1. "전체 구조 파악하자"
2. "[프로젝트/모듈] 학습하자" 또는 "work-log.md 파일 학습하자"
3. "[작업 내용] 작업 시작하자"
```

**예시**:
```
1. "전체 구조 파악하자"
2. "kj 모듈 학습하자"
3. "KJ 대리운전 검색 기능 개선 작업 시작하자"
```

---

### 방법 2: 한 줄 지시 (간단한 작업)

**기존에 작업하던 내용을 이어서 하거나, 간단한 작업일 때**

```
"[작업 내용] 작업 시작하자"
```

**예시**:
```
"약국배상 Applications 페이지 UI 개선 작업 시작하자"
"work-log.md 파일 업데이트하자"
```

---

### 방법 3: 모듈 학습 후 작업

**특정 모듈을 처음 작업할 때**

```
1. "[모듈명] 모듈 학습하자"
2. "[작업 내용] 작업 시작하자"
```

**예시**:
```
1. "약국배상 모듈 학습하자"
2. "일별실적 조회 기능 추가 작업 시작하자"
```

---

## 📋 작업 지시 패턴

### 패턴 1: 프로젝트 레벨 작업
```
"전체 구조 파악하자"
→ "[프로젝트명] 작업 시작하자"
→ "[구체적인 작업 내용]"
```

### 패턴 2: 모듈 레벨 작업
```
"[모듈명] 모듈 학습하자"
→ "[작업 내용] 작업 시작하자"
```

### 패턴 3: 연속 작업
```
"work-log.md 파일 학습하자"
→ "[이어서 작업할 내용]"
```

## 💡 상황별 작업 지시 예시

### 상황 1: 새 프로젝트 시작
```
"전체 구조 파악하자"
"kdrive 프로젝트 작업 시작하자"
```

### 상황 2: 기존 프로젝트 이어서 작업
```
"work-log.md 파일 학습하자"
"약국배상 Applications 페이지 버그 수정 작업 시작하자"
```

### 상황 3: 새 모듈 작업
```
"kj 모듈 학습하자"
"KJ 대리운전 검색 기능 개발 작업 시작하자"
```

### 상황 4: 간단한 수정/개선
```
"약국배상 Applications 페이지 UI 개선 작업 시작하자"
```

### 상황 5: 버그 수정
```
"work-log.md 파일 학습하자"
"직원 관리 페이지 로딩 오류 수정 작업 시작하자"
```

---

## ⚠️ 주의사항

1. **명확한 목표 제시**: "작업 시작하자"로 명확히 마무리
2. **구체적인 내용**: "개선", "수정", "추가" 등 구체적인 동사 사용
3. **문맥 제공**: 필요시 간단한 배경 정보 제공
4. **단계별 진행**: 복잡한 작업은 단계별로 나누어 지시

---

## 🔄 작업 지시 → 실행 흐름

```
사용자 지시
    ↓
[1단계] 학습 단계
- 전체 구조 파악 (필요시)
- 프로젝트/모듈 문서 학습
- work-log.md 확인 (필요시)
    ↓
[2단계] 작업 시작
- 작업 내용 파악
- 필요한 코드/문서 확인
- 작업 계획 수립
    ↓
[3단계] 작업 실행
- 사용자 동의 후 작업 진행
- 단계별 보고 및 확인
    ↓
[4단계] 작업 완료
- work-log.md 업데이트 (필요시)
- 다음 작업 지시 대기
```

## 📝 작업 지시 템플릿

### 템플릿 1: 새 기능 개발
```
"[모듈명] 모듈 학습하자"
"[기능명] 개발 작업 시작하자"
```

### 템플릿 2: 버그 수정
```
"work-log.md 파일 학습하자"
"[페이지/기능명] [버그 내용] 수정 작업 시작하자"
```

### 템플릿 3: UI 개선
```
"[페이지명] UI 개선 작업 시작하자"
```

### 템플릿 4: 기능 개선/추가
```
"[모듈명] 모듈 학습하자"
"[기능명] [개선/추가 내용] 작업 시작하자"
```
